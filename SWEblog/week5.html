What's your pick-of-the-week or tip-of-the-week?

Git has a function called patch, which lets you create a patch file specifying the changes to perform on another file. The patch file can be applied to the original file, which will update to include the patch. You can also revert a patch. This way, instead of having two versions of the same file, you can simply have one version and a small patch file. Patches are also used when somebody who does not have read or write access to a repository wants to make a change. They can simply send a patch file to the owners of the code, who can then apply the patch. You can find more information here.
